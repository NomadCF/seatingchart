Eye / Visibility Mode

The eye button creates a controlled viewing mode. When Eye Mode starts, the full original chart is encrypted into session storage, the visible in-memory chart is sanitized, and the app only shows the details allowed by these settings. Exiting restores the encrypted original with the Eye PIN, Lock PIN, or current encryption key.
Saving and backups Linked save file, auto-save, backups, and save/download encryption.

Save / Download Encryption

Higher values slow password guessing but may make saves/loads slower on older devices. Existing saves/backups keep the iteration count they were created with.
The Welcome screen requires an encryption password before first use. For security, the active session key is not prefilled here after entry. Updating the password changes future local saves, linked saves, backups, and new snapshots only. Older downloaded saves/backups/snapshots remain encrypted with the password used when they were created. The key is not written into exported save files; while Lock or Eye Mode is active, it can be stored only as an encrypted wrapped key protected by the Lock/Eye PIN/password.

Settings Access

Welcome to Seating Chart Bubble Builder

🔐

Build seating charts locally, protect them from the start.

This single-file browser app helps create class seating charts with students, bubbles, zones, room layouts, notes, snapshots, printing, and backups. It runs entirely in your browser.

Browser-onlyThe app stores and loads data from this browser, downloaded files, or an optional linked save file supported by the browser.
Encrypted by defaultYour first action is creating the encryption password used for full-save encryption, local autosaves, linked saves, snapshots, and backups.
Settings protected firstThe same encryption password is required the first time Settings opens. You can disable that Settings prompt later from Settings > Security & Data after a warning confirmation.
Security steps nextAfter you create the encryption password, a second Security Setup page opens so Lock PIN, Eye PIN, auto-lock, and status checks can be configured without fighting this Welcome screen.
No password recoveryThe password is not stored as plain text and cannot be recovered from this file. Save it somewhere appropriate for your district workflow.
Already used this app? If this browser has a local save, the app tries to load it before this screen appears. You can also load a downloaded save or backup file now and continue working from it.
You cannot continue until an encryption password is set or an existing encrypted save is loaded. This keeps student data encrypted from the first local save instead of waiting for someone to remember security later, which history suggests is optimistic.
